# Plan: Mobile Responsive Sidebar (Slide-out Drawer)

## Problem

Sidebar di section Child dan Elderly menggunakan `onMouseEnter`/`onMouseLeave` untuk collapse/expand. Di mobile (touch device), ini tidak berfungsi — sidebar permanen mengambil space 72px (collapsed) atau 256px (expanded). Tidak ada hamburger menu atau drawer.

## Solution

Implementasi **slide-out drawer** pattern untuk mobile dengan breakpoint `md` (768px):

- **Mobile (< 768px):** Sidebar tersembunyi, tombol hamburger muncul di navbar. Tap hamburger → sidebar geser dari kiri dengan backdrop gelap. Tap backdrop/item → tutup drawer.
- **Desktop (>= 768px):** Sidebar berfungsi seperti sekarang (mouse hover collapse/expand).

## Implementation Detail

### Mobile Drawer CSS Pattern

```
Mobile (< md):
- Sidebar: fixed inset-y-0 left-0, z-50, w-64, translate-x-0 when open / -translate-x-full when closed
- Backdrop: fixed inset-0, bg-black/50, z-40
- Body: overflow-hidden when drawer open (prevent scroll)

Desktop (>= md):
- Sidebar: relative position, existing w-[72px] / w-64 with mouse hover
- No backdrop
- No drawer state needed
```

### Hamburger Button

Reuse the same SVG pattern from `LandingNavbar.tsx`:
- `md:hidden` — only visible on mobile
- Toggles `isMobileMenuOpen` state

### Route Change Close

In both wrappers, add `usePathname()` effect to close mobile menu on navigation:
```tsx
useEffect(() => {
  setIsMobileMenuOpen(false);
}, [pathname]);
```
