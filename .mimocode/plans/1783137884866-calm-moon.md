# Rekomendasi: Konsistensi Layout Antara Child Portal & Elderly Portal

## Context

Kamu mengerjakan **Child/Family Portal** (app/child/), temanmu mengerjakan **Elderly Portal** (belum ada di codebase ini). Saat ini ada perbedaan layout:

| Elemen | Child Portal (kamu) | Elderly Portal (teman) |
|--------|---------------------|----------------------|
| Profil user | **Navbar** (kanan atas) | **Sidebar** (bawah) |
| Sidebar | Logo + 5 menu + footer copyright | Logo + 3 menu + profil user |
| Navbar | Search + notifikasi + profil | Logo + tanggal/jam |

## Analisis & Rekomendasi

### Mana yang lebih bagus?

**Profil di Sidebar (model elderly portal)** lebih baik untuk aplikasi monitoring seperti ini. Alasannya:

1. **Konsisten antar role** — Kalau profil ada di sidebar di kedua portal, user yang pakai kedua app tidak bingung. Posisi profil selalu di tempat yang sama.

2. **LebihEfisien ruang navbar** — Navbar jadi lebih bersih. Di child portal sekarang, navbar sudah padat (search + notifikasi + profil). Kalau profil dipindah ke sidebar, navbar bisa lebih fokus.

3. **Pattern yang umum** — Banyak dashboard modern (Linear, Vercel, Notion, GitHub) menaruh profil/avatar di sidebar bawah, bukan di navbar. Ini lebih intuitif karena sidebar adalah "home base" navigasi.

4. **Mobile-friendly** — Di layar kecil, navbar容易 overflow kalau ada banyak elemen. Profil di sidebar lebih aman.

### Saran Perubahan

Pindahkan profil user dari **Navbar** ke **Sidebar** (di bawah menu navigation, sebelum/atay sebagai footer). Ini membuat kedua portal punya pola yang sama.

**Child Portal (yang perlu diubah):**
- Sidebar: Tambahkan profil user (avatar + nama + role) di bagian bawah, ganti footer copyright
- Navbar: Hapus profil user, pertahankan search + notifikasi saja
