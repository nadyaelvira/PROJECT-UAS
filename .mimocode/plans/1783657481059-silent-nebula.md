# Fitur Pairing Device dengan Kode Unik

## Alur
1. Anak buka menu sidebar "Hubungkan Device" → generate kode unik 6 karakter
2. Kode ditampilkan di layar anak (bisa disalin/dibaca)
3. Lansia buka halaman pairing di device mereka → masukkan kode dari anak
4. Jika kode cocok → pairing berhasil, status tersimpan di localStorage
5. Dashboard anak menampilkan data lansia yang sudah terhubung

## File yang Perlu Dibuat/Diubah

### 1. Tambah menu sidebar anak
**File:** `components/child/Sidebar.tsx`
- Tambah menu item "Hubungkan Device" (`child.sidebar.pairDevice`) antara Dashboard dan Notifications
- Route: `/child/pairing`
- Icon: link/chain icon

### 2. Buat halaman pairing anak
**File baru:** `app/child/pairing/page.tsx`
- Tombol "Generate Kode Unik" → buat kode 6 karakter (huruf + angka)
- Tampilkan kode dengan font besar, mudah dibaca
- Tombol "Salin Kode" → copy ke clipboard
- Status pairing saat ini (sudah terhubung / belum)
- Jika sudah terhubung → tampilkan info device yang terhubung + tombol "Putuskan"

### 3. Tambah input pairing di halaman lansia
**File:** `app/elderly/home/page.tsx` atau buat modal baru
- Di halaman elderly, tambahkan prompt/input kode pairing
- Jika belum paired → tampilkan input field untuk masukkan kode
- Jika kode cocok → tandai sebagai paired, simpan ke localStorage
- Jika sudah paired → tampilkan status "Terhubung dengan [nama anak]"

### 4. State management pairing
**File:** `components/shared/sharedStore.tsx`
- Tambah state: `pairingCode: string | null` (kode yang di-generate anak)
- Tambah state: `isPaired: boolean`
- Tambah state: `pairedWith: string | null` (nama anak yang terhubung)
- Simpan ke localStorage key: `safeelder_pairing_state`
- Fungsi: `generatePairingCode()`, `setPairingCode()`, `verifyPairingCode()`, `clearPairing()`

### 6. Flow di halaman elderly
**File:** `app/elderly/home/page.tsx`
- Jika `isPaired === false` → tampilkan card/komponen di bagian atas untuk input kode
- Jika `isPaired === true` → tampilkan status "Terhubung" di navbar atau card khusus
- Kode yang diinput dicek terhadap `pairingCode` di localStorage

## Strategi localStorage

```
safeelder_pairing_state = {
  pairingCode: "AB12CD",     // kode unik yang di-generate anak
  isPaired: false,            // status pairing
  pairedByName: "Budi"        // nama anak (dari UserProfile)
}
```

Kedua device (anak & lansia) membaca key yang sama. Karena ini demo project UAS, pairing dilakukan di browser yang sama (anak buka tab `/child/pairing`, lansia buka tab `/elderly/home`).

## Verification
1. Buka `/child/pairing` → klik Generate → kode muncul
2. Buka `/elderly/home` → jika belum paired, input field muncul
3. Masukkan kode yang sama → pairing berhasil
4. Dashboard anak menampilkan status "Terhubung"
5. Navbar lansia menampilkan status "Terhubung dengan [nama]"
6. Klik "Putuskan" → pairing terputus, status kembali ke belum terhubung
