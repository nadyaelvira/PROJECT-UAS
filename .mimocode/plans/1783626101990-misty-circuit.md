# Login Validation dengan Mock Credential

## Overview
Menambahkan validasi login pada halaman login (`app/(auth)/login/page.tsx`) menggunakan mock credential hardcoded. Jika email atau password salah, tampilkan pesan error yang jelas.

## Yang Diubah

### 1. `app/(auth)/login/page.tsx`
- Tambah state `error` (string | null) untuk menyimpan pesan error
- Tambah mock credential: `admin@sapa.com` / `password123`
- Update `handleLogin`:
  1. Validasi form kosong (email/password belum diisi) → tampilkan error "Email dan password harus diisi"
  2. Cek credential yang dimasukkan vs mock credential
  3. Jika salah → tampilkan error "Email atau password salah"
  4. Jika benar → redirect ke dashboard (atau homepage, sesuai existing pattern)
- Tambahkan tampilan error message (red box/alert) di atas form
- Tambah validasi client-side: email harus format valid, password minimal 6 karakter

### 2. `context/SapaLanguageContext.tsx`
- Tambah translation keys untuk error messages:
  - `auth.login.error.empty`: "Email dan password harus diisi" / "Email and password are required"
  - `auth.login.error.invalid`: "Email atau password salah" / "Invalid email or password"
  - `auth.login.error.emailFormat`: "Format email tidak valid" / "Invalid email format"
  - `auth.login.error.passwordMin`: "Password minimal 6 karakter" / "Password must be at least 6 characters"

## Implementasi Detail

```tsx
// State baru
const [error, setError] = useState<string | null>(null);

// Mock credential
const MOCK_EMAIL = 'admin@sapa.com';
const MOCK_PASSWORD = 'password123';

// Handle login
const handleLogin = (e: React.FormEvent) => {
  e.preventDefault();
  setError(null);

  // Validasi kosong
  if (!email.trim() || !password) {
    setError(t('auth.login.error.empty'));
    return;
  }

  // Validasi format email
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    setError(t('auth.login.error.emailFormat'));
    return;
  }

  // Validasi password minimal 6 karakter
  if (password.length < 6) {
    setError(t('auth.login.error.passwordMin'));
    return;
  }

  // Cek credential
  if (email !== MOCK_EMAIL || password !== MOCK_PASSWORD) {
    setError(t('auth.login.error.invalid'));
    return;
  }

  // Login berhasil - redirect
  router.push('/'); // atau dashboard
};

// Tampilan error (di atas form)
{error && (
  <div className="mb-4 p-3 bg-red-50 border border-red-200 rounded-xl flex items-center gap-2">
    <svg className="w-5 h-5 text-red-500 shrink-0" fill="none" stroke="currentColor" viewBox="0 0 24 24">
      <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M12 8v4m0 4h.01M21 12a9 9 0 11-18 0 9 9 0 0118 0z" />
    </svg>
    <span className="text-sm text-red-600">{error}</span>
  </div>
)}
```

## User Experience
- Error muncul di atas form dengan warna merah dan ikon warning
- Error hilang saat user mulai mengetik lagi (atau submit ulang)
- Border input berubah merah saat ada error (opsional, bisa ditambahkan)
- Setelah login berhasil, redirect ke homepage (`/`)
